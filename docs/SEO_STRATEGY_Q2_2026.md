## 5) On-Page SEO Rules

### Page-level requirements
Every core landing page should have:
- 1 clear target keyword
- 1 H1 only
- strong CTA above the fold
- internal links to related pages/blog posts
- FAQ section where appropriate
- product screenshots with descriptive alt text
- clean title + meta description written for CTR, not stuffing

### Metadata direction
| Page Type | Title Pattern |
|---|---|
| Core product | Primary keyword + value prop + brand |
| Blog post | Exact or close-match keyword + practical outcome |
| Comparison | Competitor / alternative keyword + honest comparison |
| Integration page | Workflow keyword + speed/ease benefit |

### Internal linking rules
- Every blog post links to at least 2 product pages
- Every comparison page links to pricing + signup path
- Every asset-specific page links back to core journal page
- Use descriptive anchors, not “click here”

## 6) Technical SEO Priorities

## P1 — Must do in Q2
1. **Clean structured data**
   - Keep product/schema markup valid
   - Monitor GSC for rich result issues
2. **Canonical consistency**
   - Ensure all primary pages self-canonicalize correctly
3. **Titles / descriptions audit**
   - Eliminate duplicate or weak metadata on key pages
4. **XML sitemap review**
   - Include only canonical indexable pages
5. **robots.txt review**
   - Avoid blocking important app marketing pages accidentally
6. **Core page indexability audit**
   - Confirm key landing pages are crawlable and not noindex
7. **Internal link cleanup**
   - Strengthen paths into `/journal`, `/pricing`, and comparison pages
8. **Page speed basics**
   - Optimize large images
   - Reduce unused JS on landing pages where possible
   - Protect LCP on pricing / landing pages

## P2 — Strongly recommended
1. Add FAQ schema to high-intent educational pages
2. Add breadcrumb schema where useful
3. Create dedicated OG/social images for key landing pages
4. Add more text depth to thin landing pages
5. Build a reusable SEO checklist into launch/release flow
